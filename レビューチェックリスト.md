### 共通（全般）

- [ ]  コードの目的・意図が理解できるか
- [ ]  命名が分かりやすいか（変数名・関数名）
- [ ]  不要なコメントや残骸コード（デバッグ用のログなど）がないか
- [ ]  エラーハンドリングが適切にされているか
- [ ]  重複したコードがないか
- 案件別
- [ ] POST値のバリデーションチェックが行われていること
- [ ] 検証用コード(var_dump,console.log)が残っていないこと

---

### ビジネスロジック

- [ ]  仕様・要件通りの動きをするか
- [ ]  境界値・異常値のケースが考慮されているか（空文字、null、0、負の数など）
- [ ]  条件分岐の漏れがないか
- [ ]  副作用のある処理が意図した場所でだけ呼ばれているか
- [ ]  ロジックが複雑な場合、コメントや関数分割で意図が伝わるか
- 案件別
- [ ] 三項演算子は未使用になっているか

---

### DB処理（SQL）

- [ ]  N+1問題が起きていないか
- [ ]  不要なカラムをSELECTしていないか（SELECT * になっていないか）
- [ ]  インデックスが効く検索条件になっているか
- [ ]  トランザクションが必要な処理に適切に使われているか（begin,commit,rollback）
- [ ]  SQLインジェクションのリスクがないか（プレースホルダを使っているか）
- [ ]  大量データを一度に取得していないか（ページネーションなど）

---

### 画面表示

- [ ]  XSSのリスクがないか（ユーザー入力をそのまま表示していないか）
- [ ]  バリデーションがフロント・サーバー両方にあるか
- [ ]  エラー時・ローディング時の表示が考慮されているか
- [ ]  表示するデータが空の場合の考慮があるか
- [ ]  不要なAPI呼び出しや再レンダリングが起きていないか
